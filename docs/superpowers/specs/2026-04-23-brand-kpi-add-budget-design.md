# 브랜드 KPI — 매체 추가 시 예산 동시 입력

**작성일**: 2026-04-23
**대상 화면**: 브랜드 KPI 페이지 (`/admin/brands/[id]` 또는 해당 라우트)
**대상 컴포넌트**: `src/components/admin/BrandKpiManager.tsx`

## 배경

현재 브랜드 KPI 페이지에서 매체를 추가하려면:
1. "매체 추가" 버튼으로 폼을 열어 매체/매체명/시작일/종료일 입력 → 저장
2. 생성된 매체 행을 클릭해 펼친 뒤, 별도 예산 폼에서 시작일/종료일/예산을 다시 입력 → 저장

즉 **예산을 설정하려면 두 단계**를 거쳐야 하며, 예산 기간을 매체 기간과 별도로 입력해야 한다. 실제 운영상 두 기간이 다른 경우는 거의 없어 불필요한 입력이다.

## 목표

1. 매체 추가 시 **예산을 같은 폼에서 한 번에** 입력하여 매체와 예산이 동시에 생성되도록 한다.
2. **매체 기간 = 예산 기간**으로 일원화하여 사용자가 두 기간을 구분해 관리할 필요가 없게 한다.
3. 예산은 **선택(optional)** 이다 — 예산 없이 매체만 생성하는 기존 플로우를 유지한다.

## 범위 밖 (Non-goals)

- API 라우트 (`/api/admin/budget`), DB 스키마는 변경하지 않는다.
- 다른 관리자 화면(`/admin/campaigns/[id]/budget` 등)의 예산 UI는 손대지 않는다.
- 한 매체당 복수 예산 기간 관리 기능은 넣지 않는다 (현재도 1개 기준).

## 변경 사항

### 1. 매체 추가 폼

기존 4개 필드(매체 / 매체명 / 시작일 / 종료일)에 **예산 (원)** 필드를 추가한다.

- 레이아웃: 기존 `form-grid cols-4` 유지하고 예산 필드는 그 아래 한 행(cols-1 혹은 단독 블록)으로 배치. 화면 가로 폭에 맞춰 5개를 한 줄로 욱여넣지 않는다.
- 입력 타입: `type="number"`, placeholder `5000000`, optional.
- 콤마 제거 후 `parseFloat` 은 기존 `saveBudget` 로직과 동일하게 처리한다.

### 2. 저장 플로우 (`addCampaign`)

```
validate → campaigns insert
         → (예산 입력됐으면) budget POST
         → 상태 업데이트 + 폼 초기화 + 패널 닫기 + router.refresh()
```

**검증 규칙**:
- "예산 입력됨" 의 정의: 문자열 트림 결과가 빈 문자열이 아님. 즉 `0` 도 유효한 입력값으로 취급(사용자가 의도적으로 0원 예산을 기록할 수 있음).
- 예산이 입력되었는데 **종료일이 비어 있으면** 에러: `"예산 입력 시 종료일을 설정해주세요."`
  - 이유: `budgets.period_end` 는 NOT NULL.
- 예산 값이 **음수이거나 NaN** 이면 에러: `"예산은 0 이상의 숫자로 입력해주세요."`
- 예산이 비어 있으면 budget 생성을 스킵하고 매체만 생성한다.

**부분 실패 처리**:
- campaigns insert 성공 → budget POST 실패: 매체는 이미 생성되었으므로 campaigns 상태/목록에는 반영하고, 폼은 **닫고 초기화** 한다 (같은 매체가 중복 생성되는 것을 막기 위함). 안내 메시지는 폼 위가 아닌 목록 상단에 한 번 표시: *"매체는 추가되었지만 예산 저장에 실패했습니다: {error}. 해당 매체 행을 펼쳐 다시 저장해주세요."*
- campaigns insert 실패: 기존 동작과 동일(에러 표시, 폼 유지).

### 3. 확장 패널 (예산 설정)

매체 기간 = 예산 기간으로 일원화했으므로 확장 패널에서 **기간 입력은 제거**한다.

- 남기는 요소:
  - **예산 (원)** 단일 입력 필드
  - 저장 버튼 (`예산 저장` / `예산 수정`)
  - 예산 삭제 버튼 (기존 유지)
  - 이 매체 삭제 버튼 (기존 유지)
- 상단 안내문: 기존 `(period_start ~ period_end · total_budget)` 표시는 매체 `start_date ~ end_date` 기준으로 렌더. 예산 amount 는 별도 문구로 표시.
  - 예: `기간: 2026-04-01 ~ 2026-04-30 (매체 기간 기준) · 현재 예산: 1,500,000원`
- 매체에 `end_date` 가 없을 때:
  - 예산 저장 버튼 **비활성화**
  - 안내: `"이 매체의 종료일을 먼저 설정해야 예산을 저장할 수 있습니다."` (매체 종료일 편집 UI 자체는 이 스펙 범위 밖)
- `saveBudget(campaignId)` 는 캠페인 목록에서 해당 row의 `start_date`/`end_date` 를 읽어 `period_start`/`period_end` 로 사용한다.

### 4. `toggleExpand` 초기화 동작

확장 시 budgetForm 초기화 로직에서 기간 필드를 제거하고 `total_budget` 만 세팅한다.

```ts
setBudgetForm({ total_budget: existing ? String(existing.total_budget) : "" })
```

`budgetForm` 타입에서 `period_start`, `period_end` 를 제거한다.

## 데이터 모델 (변경 없음)

- `campaigns`: `start_date`, `end_date` (end_date는 nullable)
- `budgets`: `campaign_id`, `period_start`, `period_end`, `total_budget` — 스키마 그대로.

신규 제약은 **UI 레벨에서만** 성립한다: UI는 항상 `period_* = campaign.*_date` 로 저장. 기존 데이터의 budget period가 campaign 기간과 달라도 DB/API는 그대로 수용하므로 마이그레이션은 불필요.

## 테스트 시나리오

1. 매체 추가 폼에 모든 필드 + 예산 입력 → 매체와 예산이 한 번에 생성되고 목록에 반영된다.
2. 예산을 비워두고 저장 → 매체만 생성되고 "예산 미설정" 표시.
3. 예산을 입력했는데 종료일을 비움 → 에러 메시지, 저장되지 않음.
4. 매체 행을 펼친 후 예산만 수정 → 기간은 매체 기간 유지, 금액만 업데이트.
5. 종료일이 없는 기존 매체를 펼침 → 예산 저장 버튼 비활성, 안내 문구 노출.
6. 예산 삭제 → 예산만 삭제되고 매체는 유지.
